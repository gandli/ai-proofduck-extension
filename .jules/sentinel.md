## 2025-06-08 - Plaintext Storage of API Keys in Chrome Local Storage
**Vulnerability:** API keys for the various LLM providers (e.g. OpenAI, Claude, DeepSeek) were stored in Chrome Local Storage in plain text via the `zustand/middleware` `persist` plugin.
**Learning:** Even though `chrome.storage.local` is generally secure from content scripts on websites, it is accessible to any part of the extension itself. Storing sensitive credentials like API keys in plain-text without any obfuscation or encryption within the extension storage poses a significant security risk, especially in an environment where other extension scripts could inadvertently leak or log the data, or if the extension permissions are broad.
**Prevention:** Always implement at minimum Base64 encoding/obfuscation, and ideally full encryption, before persisting sensitive data like API keys into extension storage. Because the settings state often contains Unicode characters, proper Unicode encoding must be applied before Base64 encoding to prevent `InvalidCharacterError` exceptions.
