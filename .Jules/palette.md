## 2025-02-28 - Icon-only buttons missing comprehensive accessibility model
**Learning:** In this extension, several custom interactive elements (icon-only buttons in Header/SettingsPanel/LanguageSelector and custom switches) lacked a complete accessibility model. When using `focus:outline-none` to override default browser focus styles (or relying on default generic focus styles that are not distinct enough), custom controls need explicit keyboard focus indicators like `focus-visible:ring-2 focus-visible:ring-brand-orange/50`. Also, while screen readers require `aria-label`, native mouse users need a `title` attribute for tooltips on icon-only buttons.
**Action:** When building custom controls or icon-only buttons, always ensure the trio of `title`, `aria-label`, and `focus-visible:ring-2` (when `focus:outline-none` is used) are present to support both keyboard navigability and comprehensive tooltip feedback.
