# 校对鸭 BDD 清单

这份清单把浏览器扩展当前最重要的用户场景，按优先级分成 `P0 / P1 / P2`。

## P0 必测

1. 选区悬停翻译  
   选中文字后悬停 `🐣`，页内翻译卡片出现，侧边栏同步拿到同一份结果，而且不会重复翻译。

2. 点击 `🐣` 送入侧边栏  
   点击后，选区内容进入侧边栏翻译页；如果刚刚已经悬停翻过，直接复用结果，不再请求第二次。

3. 翻译卡片关闭行为  
   点关闭、点页面空白时卡片会消失；点 `Copy` 只复制，不关闭卡片。

4. 导入选区  
   侧边栏点击 `导入选区`，原文框能拿到当前网页选区。

5. 抓取全文  
   侧边栏点击 `抓取全文`，原文框能拿到整页全文。

6. 五种模式可用  
   翻译、摘要、校对、润色、扩写都能产出真实结果，不是空白占位。

7. 在线接口未配置提示  
   用户切到在线接口但没配置完整时，要给明确提示，不能静默失败。

8. 引擎来源显示  
   结果成功后，要能看出这次到底用了哪条路，比如浏览器 AI、本地 AI、在线 AI、翻译服务。

## P1 应测

9. 首选策略切换  
   切到浏览器 AI、本地模型、在线 LLM API、自动优先时，实际使用路径要跟着变。

10. 自动回退链路  
    前一条引擎不可用时，能自动退到下一条，并且用户能看懂现在走的是哪条路。

11. WebGPU / WASM 分支  
    本地模型开启后，支持 WebGPU 时走 GPU；不支持但允许兼容模式时走 WASM。

12. 策略切换后不复用旧结果  
    同一段文字，刚才用在线接口翻过；切到本地模型后，不能继续拿旧结果冒充新结果。

13. 设置持久化  
    改过目标语言、首选策略、在线接口、本地模型后，重新打开侧边栏还在。

14. 侧边栏未打开时点击 `🐣`  
    即使当时侧边栏没开，内容也要先保存，等用户打开时还能恢复出来。

15. 清空按钮  
    清空原文不影响结果；清空结果不影响原文。

16. 页内卡片与侧边栏一致  
    两边看到的结果和引擎来源要一致，不能一边说在线，一边说本地。

## P2 可补充

17. 新选区重置状态  
    上一段翻译成功变成 `🐥` 后，重新选一段新文字，要恢复成 `🐣`。

18. 连续悬停同一段  
    反复悬停同一段文字，不应该反复请求。

19. 热更新后恢复  
    开发模式内容脚本热更新后，悬停翻译仍然可用，不一直报隐藏页无响应。

20. 无选区时导入选区  
    点了也不报错，不误带旧内容。

21. 抓取全文失败  
    某些页面抓不到全文时，要有正常反馈。

22. 空文本和超短文本  
    空白、单词、短句都要行为稳定。

23. 多目标语言  
    切到 `中文 / English / 日本語` 时，结果和限制提示都正确。

24. 浏览器 AI 成功分支  
    明确选择浏览器 AI 时，要真的走浏览器 AI，并显示对应来源。

25. 浏览器 AI 语言限制  
    明确选择浏览器 AI 但目标语言不支持时，要明确失败，不能偷偷切到别的路线。

26. 本地 GPU 成功分支  
    明确选择本地模型且可用 GPU 时，要显示本地 GPU 结果与来源。

27. 本地兼容模式分支  
    明确选择本地模型且只有兼容模式时，要显示本地兼容结果与来源。

28. 页内卡片本地 GPU 分支  
    页内翻译卡片在本地 GPU 策略下，要显示本地 GPU 结果，并同步到侧边栏。

29. 页内卡片本地兼容分支  
    页内翻译卡片在本地兼容策略下，要显示本地兼容结果，并同步到侧边栏。

30. 抓取全文失败反馈  
    抓取全文失败时，要明确提示用户，并保留原来的内容。

31. 导入选区失败反馈  
    导入选区失败时，要明确提示用户，并保留原来的内容。

32. 官方模型推荐立即可见  
    打开设置页本地模型区块时，要直接看到官方模型总数和推荐模型，不应一直卡在加载提示。

33. 侧边栏已打开时复用本地模型  
    当侧边栏已经打开并且本地模型已可用时，页内翻译卡片要直接复用侧边栏那份本地结果，不能继续提示“仍在准备中”。
