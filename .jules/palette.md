## 2024-05-19 - Improved Keyboard Accessibility for Interactive Elements
**Learning:** This application had a consistent pattern of using custom styling for inputs/selects (e.g. `focus:ring-brand-orange`), but many buttons across the application (Header settings, TabBar tabs, submit buttons, swap buttons) lacked explicit focus rings when navigated via keyboard, making it difficult for keyboard-only users to determine their current position.
**Action:** When implementing custom styling on interactive elements in Tailwind projects, ensure all interactive elements (buttons, links, form fields) explicitly define `focus-visible:` utilities, typically combining `focus-visible:outline-none` with custom project ring colors (`focus-visible:ring-2 focus-visible:ring-brand-orange/50`) or matching the element's context (e.g. `focus-visible:ring-white/50` on colored backgrounds) to ensure a visible and accessible focus state without affecting mouse users.
