## 2024-04-27 - Added `type="button"` and missing `aria-label` to custom injected DOM overlay buttons
**Learning:** Found several purely visual interactive buttons in custom injected DOM overlays (like `TranslationResultLayer.tsx`) lacking explicitly defined `type="button"` and sometimes missing an `aria-label` or using only a `title`. This is important because without `type="button"`, they might act as submit buttons in a form context, and visual-only icons lack context for screen readers.
**Action:** When auditing or adding buttons in components that bypass standard design systems, explicitly ensure they have `type="button"` and appropriate `aria-label` attributes. Wrap visual characters in `<span aria-hidden="true">` inside icon-only buttons if the button has an `aria-label`.
