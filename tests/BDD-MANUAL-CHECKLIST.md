# 校对鸭手工回归打勾表

这份表是给手工回归时直接打勾用的。

建议在最新构建上执行：

1. `bun run build`
2. 装载最新扩展
3. 打开一个普通网页开始回归

---

## 1. 页内翻译入口

- [ ] 选中一段文字后，页面附近会出现 `🐣`
预期：图标位置靠近刚才操作位置，不挡住主要内容。

- [ ] 悬停 `🐣` 后，会出现小号翻译卡片
预期：卡片能完整显示，不被页面边缘裁掉。

- [ ] 悬停翻译成功后，`🐣` 会变成 `🐥`
预期：能明显看出这段内容已经翻过。

- [ ] 点击翻译卡片右上角关闭
预期：卡片消失。

- [ ] 点击翻译卡片里的 `Copy`
预期：卡片消失，且剪贴板里是译文。

- [ ] 点击页面空白处
预期：翻译卡片消失。

- [ ] 同一段文字连续多次悬停
预期：不会明显重复请求，不会反复闪烁重翻。

- [ ] 换选另一段新文字
预期：图标恢复成 `🐣`，不会错误沿用上一段状态。

- [ ] 清空选区
预期：`🐣 / 🐥` 会消失。

- [ ] 点击 `🐣`
预期：选区内容进入侧边栏，并自动切到翻译页；如果刚才已悬停翻过，不会再重复翻译。

---

## 2. 侧边栏基础操作

- [ ] 打开侧边栏后，能看到 `SOURCE`、结果区、主操作按钮
预期：布局完整，没有明显错位或滚动异常。

- [ ] 点击 `导入选区`
预期：当前页面选区会进入原文框。

- [ ] 点击 `抓取正文`
预期：当前页面正文会进入原文框。

- [ ] 依次切换 `SUMMARY / CORRECTION / POLISH / TRANSLATION / EXPANSION`
预期：5 个模式都能产出真实结果，不是空白占位。

- [ ] 点击原文区 `清空`
预期：只清原文，不清结果。

- [ ] 点击结果区 `清空`
预期：只清结果，不清原文。

- [ ] 关闭侧边栏，再重新打开
预期：设置还在；如果刚才点过 `🐣`，对应内容还能恢复。

---

## 3. 设置与引擎切换

- [ ] 打开设置，切换首选策略到 `浏览器 AI`
预期：只出现这条策略需要的设置项，不出现无关大段说明。

- [ ] 切到 `本地模型`
预期：能看到本地模型相关设置。

- [ ] 切到 `在线 LLM API`
预期：能看到在线接口相关设置。

- [ ] 切回 `自动优先`
预期：本地模型、在线接口、翻译服务相关设置会按当前策略正常出现。

- [ ] 在相同文本上切换策略后再次执行翻译
预期：会按新策略重新出结果，不会冒用旧结果。

- [ ] 结果成功后查看标题
预期：`TRANSLATION` 或其他结果标题后面会显示实际使用的引擎。

---

## 4. 浏览器 AI、本地模型、在线接口

- [ ] 明确选择 `浏览器 AI`，并执行一次支持语言的翻译
预期：结果能成功出来，并显示 `浏览器 AI`。

- [ ] 明确选择 `浏览器 AI`，目标语言改成当前不支持的语言
预期：明确失败，不会悄悄改走别的路线。

- [ ] 明确选择 `本地模型`，在 GPU 可用时执行
预期：结果成功，并显示本地 GPU 来源。

- [ ] 明确选择 `本地模型`，在只有兼容模式时执行
预期：结果成功，并显示本地兼容来源。

- [ ] 明确选择 `在线 LLM API`，并填写正确配置后执行
预期：结果成功，并显示在线 AI 来源。

- [ ] 明确选择 `在线 LLM API`，但不填写完整配置
预期：会给清楚提示，不会静默失败。

---

## 5. 回退与失败反馈

- [ ] 在线接口失败时执行翻译
预期：会自动改走翻译服务，并显示新的实际引擎。

- [ ] 手动把所有可用引擎都关掉后执行
预期：会明确失败，不会残留旧结果冒充成功。

- [ ] 当前页面没有可抓取正文时点击 `抓取正文`
预期：提示 `当前页面没有可抓取的正文`，原文框现有内容保留。

- [ ] 当前页面没有可导入选区时点击 `导入选区`
预期：提示 `当前页面没有可导入的选区`，原文框现有内容保留。
