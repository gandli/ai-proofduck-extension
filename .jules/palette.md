## 2024-05-17 - Add Clear Button to Main Input Textarea
**Learning:** Adding interactive, absolute-positioned overlay elements (like a clear button) inside a text input or textarea requires ensuring adequate padding (e.g., `pr-10` in Tailwind) on the underlying input field. Without this padding, the user's typed text will eventually overlap and render behind the interactive element, causing readability and usability issues. Furthermore, when implementing dynamic clear actions, keyboard accessibility must be maintained by immediately returning programmatic focus back to the input element via a React `useRef` (e.g., `textareaRef.current?.focus()`).

**Action:** Whenever adding floating action buttons (clear, copy, submit) inside an input container, always verify the input field has corresponding padding to reserve visual space for the button. Always ensure that interactive elements that reset or clear form state programmatically return focus to the relevant input to maintain keyboard navigation flow.
