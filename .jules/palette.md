## 2024-05-24 - Missing Explicit Focus and Titles on Custom Interactive Components
**Learning:** Custom interactive components (like custom switches built with `div`/`button` structures) and icon-only buttons frequently lack explicit keyboard focus indicators (`focus-visible:ring-2`) and `title` attributes for tooltips, which impairs accessibility. Native focus rings are sometimes completely suppressed via CSS (e.g. Tailwind `focus:outline-none`) without being properly replaced.
**Action:** Always ensure the trio of `title` (for native mouse tooltips), `aria-label` (for screen readers), and explicit keyboard focus indicators like `focus-visible:ring-2` (when standard outlines are removed) are present on custom interactive elements or icon-only buttons to support comprehensive accessibility.
